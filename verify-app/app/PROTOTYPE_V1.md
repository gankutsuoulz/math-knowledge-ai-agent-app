# 数学知识助手 APP - 原型设计 V1.0

> 整理时间：2026-05-01
> 模型：xiaomi mimo（处理基座）+ minimax（转发层）
> 目标用户：五年级～高三学生

## 一、产品概述

### 1.1 产品定位
- 产品名称：数学知识助手
- 产品类型：数学学习辅导工具
- 核心功能：拍照解题、知识点速查、例题练习、AI辅导
- 目标用户：五年级～高三学生
- 价值主张：随时随地拍题解答，AI引导自主学习

### 1.2 目标用户画像
- 主要用户：中学生（12-18岁）
- 使用场景：课后自学、作业辅导、考试复习
- 核心需求：快速解题、理解知识点、举一反三

### 1.3 核心功能矩阵
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 拍照解题 | 拍照/相册上传，AI分析解答 | P0 |
| 知识点速查 | 四大板块分类，公式渲染 | P0 |
| 例题练习 | 分知识点刷题，查看解析 | P0 |
| AI辅导 | 错误分析，引导思路 | P0 |
| 例题库管理 | 导入/导出，离线可用 | P0 |
| 收藏夹 | 收藏知识点和错题 | P1 |
| 搜索 | 关键词搜索 | P1 |

## 二、信息架构

### 2.1 底部Tab导航
- 首页（Home）
- 拍照解题（Camera）
- 知识点（Knowledge）
- 练习（Practice）
- 我的（Profile）

### 2.2 页面层级
```
首页
├── 快捷入口卡片
│   ├── 拍照解题
│   ├── 知识点速查
│   ├── 例题练习
│   └── 快捷链接到"我的"
│
拍照解题页
├── 相册选择
├── 相机拍摄
├── 框选工具
└── AI分析结果页（拍照解题流程子页面）

知识点速查页
├── 板块选择
├── 年级选择
├── 知识点列表
└── 知识点详情页

例题练习页
├── 知识点选择
├── 难度选择
├── 题目作答
└── 答案解析

我的（Profile）
├── 收藏夹页
│   ├── 收藏的知识点
│   └── 错题本
└── 设置页
    ├── API配置
    ├── 例题库导入/导出
    └── 关于
```

### 2.3 页面流转图
```
[首页] ←→ [拍照解题] ←→ [AI辅导结果]
          ↓
      [知识点速查] ←→ [知识点详情]
          ↓
        [例题练习] ←→ [答案解析]
          ↓
        [我的] ←→ [收藏夹]
          ↓
        [设置]
```

### 2.4 关键路径说明
- **拍照解题路径**：首页 → 拍照/相册 → 框选题目 → 提交 → AI分析 → 辅导结果
- **知识点查询路径**：知识点Tab → 选择板块 → 选择年级 → 浏览知识点 → 查看详情
- **例题练习路径**：练习Tab → 选择知识点 → 选择年级和难度 → 开始答题 → 查看解析 → 收藏

### 第七章：技术选型

#### 7.1 整体架构
```
┌─────────────────────────────────────────────┐
│              Android 客户端                  │
├─────────────────────────────────────────────┤
│ UI层：Jetpack Compose（声明式UI）            │
│ 业务层：ViewModel + UseCase                  │
│ 数据层：Repository + Room                    │
│ 网络层：Retrofit + OkHttp                    │
│ AI层：直接调用（APP直调，非OpenClaw）        │
└─────────────────────────────────────────────┘
                    ↓ 直接调用
┌─────────────────────────────────────────────┐
│              AI 服务层（外部）               │
│ 图像识别：mimo-v2.5（用户配置API Key）       │
│ 文字推理：mimo-v2.5-pro 或 deepseek-v4-pro  │
│           （用户可切换，API Key用户自配）    │
└─────────────────────────────────────────────┘
```

#### 7.2 技术栈明细

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 语言 | Kotlin | 1.9+ | 主开发语言 |
| UI框架 | Jetpack Compose | 1.5+ | 声明式UI |
| 最小SDK | API 26 | Android 8.0 | 兼容性 |
| DI | Hilt | 2.48+ | 依赖注入 |
| 数据库 | Room | 2.6+ | 本地存储 |
| 网络 | Retrofit + OkHttp | 4.x | REST API |
| 图片加载 | Coil | 2.5+ | 图片预览/缓存 |
| 公式渲染 | KaTeX | 0.16+ | LaTeX渲染 |
| JSON解析 | kotlinx.serialization | 1.6+ | 数据序列化 |
| 协程 | Kotlin Coroutines | 1.7+ | 异步处理 |

#### 7.3 为什么选这些技术

- **Jetpack Compose**：Google官方推荐，声明式UI开发效率高，与Kotlin完美契合
- **Room**：SQLite封装，支持Kotlin协程，与Compose配合良好
- **KaTeX**：性能优于MathJax，渲染速度快，支持离线
- **Hilt**：Google官方DI解决方案，与Android生命周期绑定良好
- **Coil**：Kotlin-first，轻量级，支持Compose

#### 7.4 替代方案对比

| 技术 | 替代方案 | 选择原因 |
|------|---------|-----------|
| Compose | XML布局 | Compose更现代，声明式更高效 |
| Room | SQLite直接使用 | Room提供协程支持和类型安全 |
| KaTeX | MathJax | KaTeX渲染性能更好 |
| Retrofit | OkHttp直接请求 | Retrofit提供更好的API抽象 |

#### 7.5 AI直调说明
- **APP直接调用AI大模型**，不经过OpenClaw转发
- **图像识别/理解**：mimo-v2.5（API Key用户在设置页填入）
- **文字推理**：mimo-v2.5-pro 或 deepseek-v4-pro（用户可切换，API Key用户自配）
- 用户在设置页配置API Key（本地加密存储）
- 标准题库更新时，用户个性化题库不被覆盖
- 收藏的题目被移除时仍保留在APP中
