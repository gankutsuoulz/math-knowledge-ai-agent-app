# 数学知识助手 APP - 整理需求文档

> 整理时间：2026-05-01
> 基于：dim1-formula.md（维度一知识体系）+ APP_PROTOTYPE.md（原型设计）
> 模型架构：xiaomi mimo（处理基座）+ minimax（转发层）
> 目标用户：五年级～高三学生

---

## 一、需求分类汇总

### 1.1 功能需求

| 编号 | 功能模块 | 功能描述 | 优先级 |
|------|---------|---------|--------|
| F-01 | 拍照解题 | 拍照/相册获取图片，框选区域，AI分析解答 | P0 |
| F-02 | 知识点速查 | 按板块/年级浏览知识点，LaTeX公式渲染 | P0 |
| F-03 | 例题练习 | 分知识点刷题，查看解析与答案 | P0 |
| F-04 | 例题库管理 | 离线批量生成例题库，导入/导出功能 | P0 |
| F-05 | 收藏夹 | 收藏知识点和错题，方便复习 | P1 |
| F-06 | 搜索功能 | 关键词搜索知识点和例题 | P1 |
| F-07 | 错误辅导 | AI分析解题错误，引导自主解决 | P0 |
| F-08 | 拍照转公式 | 从图片中提取LaTeX公式并编辑 | P2 |

### 1.2 非功能需求

| 编号 | 需求类别 | 描述 |
|------|---------|------|
| N-01 | 性能 | 公式渲染 < 100ms，离线例题库加载 < 500ms |
| N-02 | 离线能力 | 知识点、例题库完全离线可用；AI功能需联网 |
| N-03 | 存储 | 例题库采用SQLite/Room本地存储，预估 ≤ 50MB |
| N-04 | 兼容性 | Android 8.0+（API 26），Kotlin + Jetpack Compose |
| N-05 | 可扩展性 | 未来支持"维度二/维度三"知识体系，架构需解耦 |
| N-06 | 安全性 | 用户API Key加密存储，不明文传输 |

### 6.1 知识点ID编码规则

```
KB_{板块}_{年级}_{序号}

板块编码：
  A = 数与代数（Algebra）
  G = 图形与几何（Geometry）
  P = 统计与概率（Probability）
  S = 综合与实践（Synthesis）

年级编码：
  G5 = 五年级
  G6 = 六年级
  G7 = 初一
  G8 = 初二
  G9 = 初三
  G10 = 高一
  G11 = 高二
  G12 = 高三

示例：
KB_A_G9_K001  →  数与代数-初三-第001个知识点（一元二次方程一般形式）
KB_G_G8_K015  →  图形与几何-初二-第015个知识点（平行四边形判定）
```

### 6.2 难度等级定义

| 等级 | 描述 | 适用场景 |
|------|------|---------|
| 基础 | 直接套用公式/定理，1～2步完成 | 知识点初学 |
| 中等 | 需要理解概念，结合多个知识点 | 单元复习 |
| 困难 | 综合性强，需要技巧或分类讨论 | 冲刺提高 |
