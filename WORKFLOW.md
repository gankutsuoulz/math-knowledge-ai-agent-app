## 一、编码+Review流程

每个工作项执行：**mimo编码 → deepseek review → 修正 → 交付**

**通知要求：**
- 提交mimo编码时：告知"此代码将由deepseek-v4-pro进行CodeReview"
- 提交deepseek Review时：告知"此代码由mimo-v2.5-pro编写"

**API调用方式：**
- 使用 Python 直接调用 API（绕过 sessions_spawn.model 被忽略的问题）
- 编码模型：`mimo-v2.5-pro`
- Review模型：`deepseek-v4-pro`
