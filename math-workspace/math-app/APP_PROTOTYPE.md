# 数学知识助手 APP - 需求汇总与原型设计

> 整理时间：2026-05-01
> 模型架构：xiaomi mimo（处理基座）+ minimax（OpenClaw 基座，仅转发）

---

## 一、需求汇总

### 需求 1：基础框架
- **目标**：创建数学类安卓 APP
- **知识来源**：`/projects/math-workspace/dim1-formula.md`（维度一知识体系）
- **源码路径**：`/projects/math-workspace/math-app/`（单独新建，与原知识文件分开）
- **功能**：基于 dim1 整理分类知识点，并生成各知识点例题
- **例题审核**：生成后使用 **DeepSeekV4Pro** 检查，确保无错误

### 需求 2：模型架构
- **处理基座**：xiaomi mimo（负责实际项目处理）
- **转发模型**：minimax（OpenClaw 基座，仅转发用户需求给 mimo 并返回结果）

### 需求 3：知识点速查
- 实现知识点快速查阅功能
- 保证数学公式（LaTeX）正确显示

### 需求 4：拍照/图片解析 + AI 辅导
- **图片获取**：拍照或相册选择
- **框选功能**：用户对图片内容进行区域框选标注
- **图片识别**：将框选区域 + 原图上传至 **mimo 图像解析 API**
- **解题分析**：识别结果发送至 **mimo2.5Pro** 分析用户解答是否正确
- **错误辅导**（当存在错误或未完整解题时）：
  - 标记题目涉及的**数学知识点**
  - 提供该知识点**典型例题**及**解题思路**
  - 指出原题**解题引导思路**（⚠️ 非完整解题步骤）
- **功能定位**：协助辅导用户理解题目和知识点，**引导用户自主解决**，而非 AI 直接给答案

## 三、技术架构建议

| 模块 | 技术选型 |
|------|---------|
| 前端框架 | Kotlin + Jetpack Compose |
| 公式渲染 | MathView / KaTeX WebView |
| 图片框选 | Canvas 绘制 + Touch 事件 |
| AI 图像解析 | mimo 图像解析 API |
| AI 文本分析 | mimo2.5Pro API |
| 例题审核 | DeepSeekV4Pro API |
| 本地存储 | Room 数据库 |
| 图片加载 | Coil |
| 网络 | Retrofit + OkHttp |

---

## 四、API 对接说明

| 功能 | 调用方式 |
|------|---------|
| mimo 图像解析 / mimo2.5Pro | 通过 **OpenClaw** 调用，APP 无需直接对接 |
| DeepSeekV4Pro（例题审核） | 通过 **OpenClaw** 调用，APP 无需直接对接 |

> ⚠️ mimo 和 DeepSeek 的 API 对接已在 OpenClaw 侧配置完毕，本 APP 通过 OpenClaw 间接触发调用，无需单独配置。
